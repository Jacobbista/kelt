🌊 Bufferbloat (Loaded Latency)
This test evaluates how well the 5G link (dongle + UPF) manages its queues via AQM (Active Queue Management).
It runs a heavy iperf3 throughput test to saturate the link, while simultaneously sending a continuous ping.
What to watch for: If the red RTT chart (latency) spikes severely when throughput hits its peak, the link suffers from Bufferbloat. A healthy link keeps latency stable even under maximum load. Sequential mode tests this in both DL and UL directions automatically.
📻 Before you run — check the radio
Open the dongle WebUI (see Namespaces panel), go to Status → Network and verify:
  • Mode: must be 5G NR (not 4G fallback)
  • RSRP: ideally > −100 dBm (−80 dBm = excellent)
  • RSSI / RSRQ: stable between runs — if this fluctuates, it explains throughput variance, not bufferbloat
Note the values before each test run for your thesis log.
